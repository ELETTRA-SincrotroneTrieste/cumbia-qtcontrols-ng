\doxysubsection{Detailed Description}
The main class that contains the plot canvas. 

This library can draw curves on a plot canvas. Each curve is represented by a \doxyref{Scene\+Curve}{p.}{classSceneCurve}. A \doxyref{Scene\+Curve}{p.}{classSceneCurve} contains the data of the curve. The \doxyref{Scene\+Curve}{p.}{classSceneCurve} is not responsible for drawing its shape. Each \doxyref{Scene\+Curve}{p.}{classSceneCurve} must be given a name and added to the plot by means of \doxyref{QGraphics\+Plot\+Item\+::add\+Curve}{p.}{classQGraphicsPlotItem_aca6d96b3129ef86ea06ff81fa1833de8} ~\newline
 The painting takes place by a \doxyref{Curve\+Item}{p.}{classCurveItem} object constructed by passing a \doxyref{Scene\+Curve}{p.}{classSceneCurve} to it and by an implementation of an \doxyref{Item\+Painter\+Interface}{p.}{classItemPainterInterface} which must be installed on a \doxyref{Curve\+Item}{p.}{classCurveItem}.

The \doxyref{Scale\+Item}{p.}{classScaleItem} class represents an axis scale. More than a couple of axes may be added to the plot. \doxyref{Axes\+Manager}{p.}{classAxesManager} class is responsible of coupling and managing the axes.

The \doxyref{Target\+Item}{p.}{classTargetItem} item is a QGraphics\+Object that can be moved over the plot and it is aimed at realizing writers.

Readers that want to draw curves on the plot have to conform to the \doxyref{XYPlot\+Interface}{p.}{classXYPlotInterface} interface that the \doxyref{QGraphics\+Plot\+Item}{p.}{classQGraphicsPlotItem} implements.

In the {\itshape examples} directory you will find some examples of the usage of the library.

\begin{DoxyParagraph}{Properties and settings}
Each QObject (in particular QGraphics\+Object for the items in a plot) can export its relevant properties through a set of {\itshape Q\+\_\+\+PROPERTY} declarations. These properties are then taken into account at runtime and the most important types of properties can be modified and saved. Each object will display its properties under a page of a tab widget. To register QObject\textquotesingle{}s properties, the object must have an object\+Name and you must provide a settings key through the set\+Settings\+Key method call, passing a string as a key for the properties of your objects to be saved by the QSettings class. Normally, you want to pass the application name as settings key.
\end{DoxyParagraph}
\begin{DoxyNote}{Note}
Since version 2.\+2.\+2, Qt {\itshape dynamic properties} directly set on QObjects with {\itshape set\+Property} are taken into account and made available by the configuration dialog. 

Dynamic properties starting with two underscores {\itshape \char`\"{}\+\_\+\+\_\+\char`\"{}} are reserved and do not have to be used. 

Since dynamic properties are expected to be application specific, their names are put in evidence by means of a bold font in the configuration dialog.
\end{DoxyNote}

\begin{DoxyParagraph}{Position of the axes}
The axes are placed by \doxyref{QGraphics\+Plot\+Item}{p.}{classQGraphicsPlotItem} through the classes \doxyref{Axes\+Manager}{p.}{classAxesManager} and \doxyref{Axis\+Couple}{p.}{classAxisCouple}. Each \doxyref{Axis\+Couple}{p.}{classAxisCouple} in the \doxyref{QGraphics\+Plot\+Item}{p.}{classQGraphicsPlotItem} represents a couple of axis, one with Horizontal orientation and the other with Vertical orientation. The default positioning of the axes is done by default such as each the axes cross each other in the middle. If you want to change the reciprocal axes position, see \doxyref{QGraphics\+Plot\+Item\+::set\+Origin\+Pos\+Percentage}{p.}{classQGraphicsPlotItem_af682c855baf92e2c96d0ed6b6ca747ba} \doxyref{QGraphics\+Plot\+Item\+::set\+Default\+XAxis\+Origin\+Pos\+Percentage}{p.}{classQGraphicsPlotItem_a77a879f47f39475a989da91bf8c8c700} and \doxyref{QGraphics\+Plot\+Item\+::set\+Default\+YAxis\+Origin\+Pos\+Percentage}{p.}{classQGraphicsPlotItem_a38216c1b91b437092a5f7c73fbfb377c}.
\end{DoxyParagraph}

\begin{DoxyParagraph}{Plot\+Scene\+Widget}
\doxyref{Plot\+Scene\+Widget}{p.}{classPlotSceneWidget} is a QGraphics\+View with a QGraphics\+Scene where QGraphics\+Item items can be placed and displayed. In particular, \doxyref{Plot\+Scene\+Widget}{p.}{classPlotSceneWidget} is a {\itshape plot} where {\itshape curves} can be drawn.

\begin{DoxyParagraph}{Axes}
A \doxyref{Plot\+Scene\+Widget}{p.}{classPlotSceneWidget} has by default a couple of axes, the x and y axes, and their origin is by default in the center of the scene. If you want to change the axes position, see \begin{DoxyItemize}

\end{DoxyParagraph}
\begin{DoxyParagraph}{Reducing the plot area inside the scene.}
It is possible to reduce the area occupied by the plot by means of the plot\+Area\+Percentage\+Width, plot\+Area\+Percentage\+Height, plot\+Area\+Top\+Left\+XPercentage, plot\+Area\+Top\+Left\+YPercentage properties.
\end{DoxyParagraph}
\begin{DoxyParagraph}{Curves}
Each curve is represented by a \doxyref{Scene\+Curve}{p.}{classSceneCurve}. A \doxyref{Scene\+Curve}{p.}{classSceneCurve} contains the data of the curve. The \doxyref{Scene\+Curve}{p.}{classSceneCurve} is not responsible for drawing itself. Each \doxyref{Scene\+Curve}{p.}{classSceneCurve} must be given a name and added to the plot by means of \doxyref{Plot\+Scene\+Widget\+::add\+Curve}{p.}{classPlotSceneWidget_ae3bf64dfa4b9c8e2d3bf5ff8bfdc2a9e} ~\newline
 The painting takes place by a \doxyref{Curve\+Item}{p.}{classCurveItem} object instantiated by passing a \doxyref{Scene\+Curve}{p.}{classSceneCurve} to its constructor and by an implementation of an \doxyref{Item\+Painter\+Interface}{p.}{classItemPainterInterface} which must be installed on a \doxyref{Curve\+Item}{p.}{classCurveItem}. Once a \doxyref{Curve\+Item}{p.}{classCurveItem} is constructed with a \doxyref{Scene\+Curve}{p.}{classSceneCurve}, the \doxyref{Scene\+Curve}{p.}{classSceneCurve} {\itshape takes the ownership of the \doxyref{Curve\+Item}{p.}{classCurveItem}, destroying it when the curve is destroyed}. Once an \doxyref{Item\+Painter\+Interface}{p.}{classItemPainterInterface} implementation instance is installed on a \doxyref{Curve\+Item}{p.}{classCurveItem}, the \doxyref{Curve\+Item}{p.}{classCurveItem} {\itshape takes the ownership of the item painter} destroying it on \doxyref{Curve\+Item}{p.}{classCurveItem} destruction.
\end{DoxyParagraph}
\doxyref{Line\+Painter}{p.}{classLinePainter} is an example of an \doxyref{Item\+Painter\+Interface}{p.}{classItemPainterInterface} implementation. \doxyref{Dots\+Painter}{p.}{classDotsPainter} is another example.

\begin{DoxyParagraph}{Why ownership of Curve\+Item and Line\+Painter is passed to Scene\+Curve and Curve\+Item respectively?}

\begin{DoxyParagraph}{Some more details about item deletion.}
\begin{DoxyItemize}
\item {\itshape \doxyref{Curve\+Item}{p.}{classCurveItem}} is a QGraphics\+Object and when it\textquotesingle{}s added to a QGraphics\+Scene through QGraphics\+Scene\+::add\+Item the ownership is taken by the QGraphics\+Scene. This means that every \doxyref{Curve\+Item}{p.}{classCurveItem} is deleted on scene deletion, i.\+e. on \doxyref{Plot\+Scene\+Widget}{p.}{classPlotSceneWidget} deletion, since the plot scene is a child of \doxyref{Plot\+Scene\+Widget}{p.}{classPlotSceneWidget}. \item {\itshape \doxyref{Scene\+Curve}{p.}{classSceneCurve}} is created by \doxyref{Plot\+Scene\+Widget\+::add\+Curve}{p.}{classPlotSceneWidget_ae3bf64dfa4b9c8e2d3bf5ff8bfdc2a9e} as a child of \doxyref{Plot\+Scene\+Widget}{p.}{classPlotSceneWidget}. Each curve is thus deleted on \doxyref{Plot\+Scene\+Widget}{p.}{classPlotSceneWidget} destruction. \item {\itshape \doxyref{Line\+Painter}{p.}{classLinePainter}}, as in the {\itshape example above}, must be constructed by passing a \doxyref{Curve\+Item}{p.}{classCurveItem} in its constructor. {\itshape All \doxyref{Item\+Painter\+Interface}{p.}{classItemPainterInterface} implementations}, such as \doxyref{Line\+Painter}{p.}{classLinePainter}, must inherit from QObject and the parent must be a \doxyref{Curve\+Item}{p.}{classCurveItem}, as you can see in the \doxyref{Line\+Painter}{p.}{classLinePainter} implementation. In this way, on \doxyref{Curve\+Item}{p.}{classCurveItem} deletion, the \doxyref{Line\+Painter}{p.}{classLinePainter} is automatically deleted, as a child of \doxyref{Curve\+Item}{p.}{classCurveItem}.\end{DoxyItemize}

\end{DoxyParagraph}

{\footnotesize\ttfamily void QGraphics\+Plot\+Item\+::set\+Manual\+Update (\begin{DoxyParamCaption}\item[{bool}]{manual }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [slot]}}



This is an important optimization that acts on the QGraphics\+View\+::\+Viewport\+Update\+Mode that comes into relevance when you have several items that must be fastly drawn. 

This property sets the Viewport\+Update\+Mode to {\bfseries{QGraphics\+View\+::\+No\+Viewport\+Update}} meaning that QGraphics\+View will never update its viewport when the scene changes; the user is expected to control all updates. This mode disables all (potentially slow) item visibility testing in QGraphics\+View, and is suitable for scenes that either require a fixed frame rate, or where the viewport is otherwise updated externally.

\doxyparagraph*{Note}

When you enable manual scene update, then you must call set\+Refresh\+Period with the desired refresh period of the scene, which typically must be equal to the fastest refresh rate needed by the fastest item. ~\newline
 Calling set\+Manual\+Scene\+Update with a false parameter, restores QGraphics\+Scene default Viewport\+Update\+Mode, which is QGraphics\+View\+::\+Minimal\+Viewport\+Update, according to the Qt documentation.


\begin{DoxyParams}{Parameters}
{\em manual} & true update is managed by an internal timer. set\+Refresh\+Period must be called with a greater than zero value in order to activate the timer and have the scene refreshed with that interval.\\
\hline
{\em manual} & false the update is set to QGraphics\+View\+::\+Minimal\+Viewport\+Update, the default Qt setting for the Viewport\+Update\+Mode property. The internal refresh timer, if previously activated and started with a valid set\+Refresh\+Period method call is {\bfseries{automatically}} stopped for you. The timer is not deleted, it is just stopped, so that a subsequent call to set\+Manual\+Scene\+Update with a true parameter {\bfseries{automatically restarts the timer with the timeout previously set with set\+Refresh\+Period}}. In this way, a second manual mode activation does not need another call to set\+Refresh\+Period. A call to set\+Refresh\+Period with zero or a negative value instead produces the {\bfseries{ destruction}} of the timer.\\
