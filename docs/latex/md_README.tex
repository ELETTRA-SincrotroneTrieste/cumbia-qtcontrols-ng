\doxysection*{Installation}

\doxysubsection*{Dependencies}

Qt libraries

\doxysubsection*{Download build and install}

\begin{quote}
git clone {\texttt{ https\+://github.\+com/\+ELETTRA-\/\+Sincrotrone\+Trieste/cumbia-\/qtcontrols-\/ng.\+git}} cd cumbia-\/qtcontrols-\/ng qmake \&\& make \&\& make install \end{quote}
\doxysubsection*{Options}

You can either edit cumbia-\/qtcontrols-\/ng.\+pri to change the {\itshape INSTALL\+\_\+\+ROOT} installation prefix or pass the {\itshape INSTALL\+\_\+\+ROOT=path/to/qumbia-\/svg} to qmake\+:

\begin{quote}
qmake INSTALL\+\_\+\+ROOT=/my/custom/dir \end{quote}
\doxysubsubsection*{Default prefix}

The default prefix is $\ast$/usr/local/cumbia-\/libs$\ast$ (includes in $\ast$/usr/local/cumbia-\/libs/include/cumbia-\/qtcontrols-\/ng/$\ast$, libs in $\ast$/usr/local/cumbia-\/libs/lib/$\ast$ and so forth)

\doxysubsection*{Important advice}

Please make sure the installation prefix is the same as that used for the {\itshape cumbia} library. This ensures proper library interoperability.
